# Contributing Guide — PHAROS Ethical Stack

Thank you for your interest in contributing to the E-IP / LDT-based Ethical Protocol Stack.  
This guide outlines how to participate effectively and responsibly.

---

## 1. Ways to Contribute

You can contribute by:
- Submitting pull requests  
- Triaging issues  
- Writing documentation  
- Proposing protocol changes (RFCs)  
- Creating diagrams and examples  
- Building tooling (validators, detectors, etc.)  
- Improving schemas or definitions  

All contributions must align with the PHAROS Ethical Stack principles.

---

## 2. Contribution Workflow

### Step 1: Open an Issue
Use GitHub Issues to describe:
- Problem or bug  
- Proposal or feature  
- Implementation notes  

### Step 2: Fork the Repository
Make changes in a feature branch.

### Step 3: Submit a Pull Request
Every PR must include:
- Clear summary  
- Motivation  
- Link to Issue or RFC  
- Ethical considerations  

### Step 4: Maintainer Review
A maintainer will:
- Review code  
- Suggest changes  
- Approve or request revision  

### Step 5: Merge
Only maintainers may merge PRs.  
Significant changes require an **RFC**.

---

## 3. RFC Requirement

You MUST submit an RFC for:
- Protocol changes  
- Spec changes  
- Schema changes  
- New layers, fields, or definitions  
- Governance changes  

RFCs follow `/RFC/TEMPLATE.md`.

---

## 4. Coding Standards

- Use clear, explicit variable names  
- Prefer declarative logic  
- Document complex logic in comments  
- Follow semantic versioning guidelines  
- Avoid breaking backward compatibility  

---

## 5. Ethical Standards

All contributions must:
- Protect human agency  
- Avoid manipulation, opacity, or coercion  
- Prevent semantic drift  
- Support transparency and traceability  
- Align with LDT-informed principles  

If a contribution raises ethical concerns, it must undergo an **Alignment Review**.

---

## 6. Commit Message Format

Use:
```
[area] summary of change
```

Examples:
```
[spec] add semantic-field definition
[tools] update manifest-validator schema support
[docs] clarify governance process
```

---

## 7. Conduct Requirements

- Be respectful  
- No harassment or hostility  
- Assume good intent  
- Focus on constructive collaboration  

---
